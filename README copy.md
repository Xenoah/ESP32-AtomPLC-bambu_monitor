# ESP32-AtomPLC-sample

M5Stack ATOM を本体にして、I2C 接続した MCP23017 と SSD1306 OLED を使い、PLC 風の入出力表示を行うサンプルです。

このプロジェクトでは次の 3 つを同時に扱います。

- M5 ATOM 本体の 5x5 LED 表示
- MCP23017 の 8 点入力 / 8 点出力
- 128x64 SSD1306 OLED への状態表示

## 概要

起動すると I2C バス上の OLED と MCP23017 を初期化し、MCP23017 の状態が取れたらループ動作に入ります。

ループ中は以下を行います。

- 1 秒ごとに 0 から 9 までカウントアップ
- その値を `Y0` から `Y3` に 4bit の二進数として出力
- `X0` から `X7` の入力状態を読み取る
- `Y0` から `Y7` の出力状態を読み取る
- M5 ATOM の 5x5 LED に入力 / 出力を表示
- OLED に `X` / `Y` / カウンタ値 / 稼働時間を表示

## ハードウェア構成

想定している構成は次の通りです。

- マイコン: M5Stack ATOM
- I/O エキスパンダ: MCP23017
- OLED: SSD1306 128x64 I2C

I2C は M5 ATOM 側で以下のピンを使っています。

- `SDA = GPIO25`
- `SCL = GPIO21`

I2C アドレスはコード上で固定です。

- MCP23017: `0x20`
- SSD1306 OLED: `0x3C`

## ピンアサイン

### M5 ATOM 側

| 用途 | GPIO | 接続先 |
| --- | --- | --- |
| I2C SDA | GPIO25 | MCP23017 SDA, OLED SDA |
| I2C SCL | GPIO21 | MCP23017 SCL, OLED SCL |
| 内蔵 5x5 LED | 内蔵デバイス | 状態表示に使用 |
| USB Serial | USB | 書き込み / シリアルモニタ |

### MCP23017 側

| 論理名 | 物理ポート | 方向 | 論理 |
| --- | --- | --- | --- |
| `X0` - `X7` | `GPA0` - `GPA7` | 入力 | `LOW` を `1` として扱う |
| `Y0` - `Y7` | `GPB0` - `GPB7` | 出力 | `HIGH` を `1` として扱う |

入力側は `INPUT_PULLUP` で初期化しているため、スイッチや接点は GND に落とす前提です。

つまり入力は以下の動きになります。

- 未接続 / 開放: `0`
- GND に接続: `1`

出力側は起動時にすべて `LOW` に初期化します。

## ソフトウェア構造

実装はほぼすべて [`src/main.cpp`](C:/GitHub/ESP32-AtomPLC-sample/src/main.cpp) に入っています。

### 1. 入出力の抽象化

`XRef` / `YRef` という小さなラッパーを使って、PLC 風に `X0`, `X1`, `Y0`, `Y1` のように扱えるようにしています。

- `X(i)` は MCP23017 の `GPAi` を読む
- `Y(i)` は MCP23017 の `GPBi` を読む / 書く
- `X0` - `X7`, `Y0` - `Y7` はマクロで定義

これにより、通常の PLC ラダーの入出力記号に近い感覚でロジックを書けます。

### 2. 表示ユーティリティ

5x5 LED の表示用に次の補助関数があります。

- `fillAll()`: 全 LED を同色で塗る
- `setDot()`: 任意座標の LED を点灯
- `drawX()`: 入力 `X` の 8bit を左側に表示
- `drawY()`: 出力 `Y` の 8bit を右側に表示

表示色は次のルールです。

- `X` 側 ON: 緑
- `Y` 側 ON: 赤
- OFF: 暗いグレー
- 中央列: ハートビート点滅

### 3. MCP23017 初期化

`try_init_mcp()` で MCP23017 を初期化します。

処理内容は次の通りです。

- `0x20` で I2C 接続を開始
- `GPB0` - `GPB7` を出力に設定
- 出力をすべて `LOW` に初期化
- `GPA0` - `GPA7` を `INPUT_PULLUP` に設定

### 4. `setup()`

`setup()` では次の順序で初期化します。

1. M5 ATOM 本体の初期化
2. 5x5 LED の明るさ設定
3. I2C 開始
4. OLED 初期化
5. MCP23017 の初期化待ち

MCP23017 の認識に失敗している間は、以下の見た目になります。

- M5 ATOM 5x5 LED: 赤点滅
- OLED: `MCP23017 ERROR`

認識に成功すると、以下になります。

- M5 ATOM 5x5 LED: 一瞬全緑
- OLED: `MCP23017 OK`

## 動作の仕組み

### カウンタ出力

`loop()` 内で 1 秒ごとに `counter` を更新し、`0` から `9` を繰り返します。

その値を 4bit の二進数として次に割り当てています。

- `Y0`: 1 の位 bit0
- `Y1`: bit1
- `Y2`: bit2
- `Y3`: bit3

例えば `counter = 5` のときは `0101` なので次の状態です。

- `Y0 = 1`
- `Y1 = 0`
- `Y2 = 1`
- `Y3 = 0`

`Y4` - `Y7` は現在のサンプルでは未使用ですが、状態読み取りと表示には含めています。

### 入力の読み取り

`X0` - `X7` は毎ループ読み取り、8bit の `xbits` にまとめています。

入力は `LOW` を `1` とみなすため、PLC 的には「接点が入ったら 1」と見やすい構成です。

### 出力の読み取り

`Y0` - `Y7` も毎ループ読み取り、8bit の `ybits` にまとめています。

これにより、実際に出している出力状態をそのまま表示できます。

### M5 ATOM 5x5 LED 表示

5x5 LED の見方は次の通りです。

- 左 2 列: `X0` - `X7`
- 中央 1 列: セパレータ兼ハートビート
- 右 2 列: `Y0` - `Y7`
- 最下段の左右: 入力 / 出力の色見本

ビット配置は上から `0` から `3`、その下に `4` から `7` を並べる形ではなく、各 4 点列を 2 列で表現しています。

具体的には次の割り当てです。

| 表示対象 | 列 | 行 | ビット |
| --- | --- | --- | --- |
| `X0` - `X3` | 0 | 0 - 3 | `X0` から順 |
| `X4` - `X7` | 1 | 0 - 3 | `X4` から順 |
| 中央列 | 2 | 0 - 3 | ハートビート |
| `Y0` - `Y3` | 3 | 0 - 3 | `Y0` から順 |
| `Y4` - `Y7` | 4 | 0 - 3 | `Y4` から順 |

### OLED 表示

OLED には以下を表示します。

- `X=` の後ろに 8bit 入力状態
- `Y=` の後ろに 8bit 出力状態
- `CNT=` の後ろに 0 から 9 の現在値
- `ms=` の後ろに `millis()`

`X` と `Y` はゼロサプレスなしで常に 8bit 表示です。

## 使いかた

### 1. 配線

最低限、次を接続してください。

- M5 ATOM `GPIO25` -> MCP23017 `SDA`
- M5 ATOM `GPIO21` -> MCP23017 `SCL`
- M5 ATOM `GPIO25` -> OLED `SDA`
- M5 ATOM `GPIO21` -> OLED `SCL`
- GND 共通
- 電源共通

I2C のプルアップ有無は使用しているモジュール構成に依存します。一般的な OLED / MCP23017 ブレークアウト基板なら、基板側に実装済みのことが多いです。

### 2. ビルド

PlatformIO を使います。

```powershell
pio run
```

### 3. 書き込み

```powershell
pio run -t upload
```

このプロジェクトでは [`platformio.ini`](C:/GitHub/ESP32-AtomPLC-sample/platformio.ini) を次のようにしています。

```ini
monitor_speed = 115200
upload_speed  = 115200
```

以前 `upload_speed = 921600` では、接続直後に通信が切れて書き込みに失敗しました。現状は `115200` が安定設定です。

### 4. シリアルモニタ

```powershell
pio device monitor
```

このサンプルは現状、シリアルへ詳細ログを出していません。必要なら `Serial.begin()` と `Serial.println()` を追加してください。

## 確認ポイント

正常起動時は以下を確認できます。

- OLED に `MCP23017 OK` が出たあと、`X=` `Y=` `CNT=` `ms=` が更新される
- 5x5 LED の中央列が点滅する
- `Y0` - `Y3` が 1 秒ごとにカウントに応じて変化する
- 入力を GND に落とすと該当する `X` ビットが点灯する

## トラブルシュート

### MCP23017 ERROR のまま進まない

次を確認してください。

- I2C 配線が `GPIO25 / GPIO21` になっているか
- MCP23017 のアドレスが `0x20` か
- GND が共通になっているか
- 電源電圧がモジュール仕様に合っているか

### OLED が表示されない

次を確認してください。

- OLED の I2C アドレスが `0x3C` か
- OLED と MCP23017 が同じ I2C バスにつながっているか
- 電源と GND が正しいか

### アップロードに失敗する

この環境では高速書き込みが不安定でした。次を優先して確認してください。

- `upload_speed = 115200` になっているか
- USB ケーブルが充電専用でないか
- 別の USB ポートを試したか
- シリアルポートを他アプリが掴んでいないか
